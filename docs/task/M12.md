## Limiti residui espliciti
1. **Differenze locale vs EC2**: latenza, throughput e timing gossip possono differire sensibilmente.
2. **Caveat IP pubblico EC2**: dopo stop/start istanza l’IP pubblico può cambiare, impattando endpoint/documentazione operativa se hardcoded.
3. **Scope multi-host**: il percorso canonico M12 resta single-host (1 EC2 + Compose); multi-host richiede orchestrazione/networking più fragile e fuori dallo scope standard.
4. **Dipendenza ambientale Docker/Compose**: la validazione end-to-end continua a dipendere dalla disponibilità del runtime container locale/EC2.

## Follow-up consigliati
1. aggiungere un playbook operativo “golden path” con checklist unica (locale + EC2) collegata agli script `scripts/cluster_*`;
2. estendere la sezione EC2 con template SG/porte precompilati per ridurre errori manuali;
3. introdurre una mini-sezione di troubleshooting specifica per cambio IP pubblico e riallineamento rapido degli endpoint;
4. valutare, in milestone successiva separata, una traccia controllata multi-host EC2 con criteri minimi di ripetibilità.
